# Thinking Before Command

Every command changes something.

This repository is a reminder that taking a few seconds to think before pressing **Enter** can prevent mistakes, reduce downtime, and protect systems.

## Why This Repository Exists

Many command-line mistakes happen because commands are executed too quickly without checking:

- The current directory
- The target system
- Command syntax
- Required permissions
- Potential consequences

A short pause before execution can prevent accidental file deletion, service outages, security incidents, and data loss.

## Think Before You Type

Before running any command, ask yourself:

- Am I on the correct machine?
- Am I in the correct directory?
- Do I understand what this command does?
- Will this command modify or delete anything?
- Do I have a backup if something goes wrong?
- Can I test this safely first?

## Golden Rule

> Think twice. Press Enter once.

## Examples

Instead of immediately running:

```bash
rm -rf *
```

Pause and verify:

- Current directory
- Files that will be removed
- Whether a backup exists

Instead of running:

```bash
git push
```

Check:

- Current branch
- Commit history
- Remote repository
- Changes being pushed

Every command changes something.

Pausing to think before pressing **Enter** prevents most mistakes.
